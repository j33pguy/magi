# MAGI v0.5.0 release checklist

## Scope freeze
- Confirm the exact release commit, currently tested from `9855bcf2fd6f78a3ed90009114a1b1d6fc626d14`.
- Decide whether the staging-only BM25 fallback patch stays out of release, is upstreamed, or is reverted before prod rollout.
- Bump the reported app version from `0.3.0` to `0.5.0` before publishing.

## Pre-release verification
- Run targeted tests:
  - `go test ./internal/remember ./internal/search ./internal/pipeline ./internal/db ./internal/server ./internal/api ./internal/tools`
- Build release binary:
  - `CGO_ENABLED=1 go build -o .tmp-release/magi ./`
- Verify redesign paths manually against a staging instance:
  - `POST /remember` succeeds with async writes enabled
  - repo facet inference adds `repo:<owner>/<repo>` tags
  - `GET /search` returns expected repo-scoped result for redesign queries
  - `GET /health` stays green after restart
- Check migration safety on the target backend in use.

## Staging rollout
- Backup current staging binary and database.
- Install the new binary.
- Restart the staging service.
- Confirm ports and health on staging (`18300/18301/18302`).
- Run targeted remember/search validation and inspect logs for startup or pipeline regressions.

## Production readiness
- Take a fresh backup of `/opt/magi/data/memory.db` and `/opt/magi/data/git-memories`.
- Confirm enough free disk for DB and git history growth.
- Confirm rollback path with prior binary still available.
- Confirm auth settings, proxy auth, and ONNX runtime env are unchanged.

## Production rollout
- Deploy during a quiet window.
- Restart `magi.service`.
- Validate:
  - `/health`
  - `/search`
  - one safe `POST /remember`
  - web UI on `:8080`
  - gRPC and gateway listeners on `:8300/:8301/:8302`
- Watch journal for 5 to 10 minutes for startup, embedding, BM25, or contradiction errors.

## Post-release
- Tag release and publish notes.
- Record deployed commit, binary hash, deploy time, and operator.
- Monitor dedup/search quality and async write behavior for the first day.
